## General Settings

Configure basic display options for your directory.

### Top Tags Display

Control how many top tags to display on your directory:

<CodeGroup>
```example "Common Configurations"
10 tags  → Standard display
15 tags  → More comprehensive view
5 tags   → Minimal, focused approach
20 tags  → Maximum visibility
```
</CodeGroup>

<Tip>
Choose a number that balances visibility with clean design. Too many tags can overwhelm visitors, while too few might limit discoverability.
</Tip>

## Badge Settings

Customize the "Built with DirectoryEasy" badge display and affiliate settings.

### Badge Display Options

<Tabs>
  <Tab title="Fixed Floating Badge">
    The badge appears as a floating element that's always visible as users scroll through your directory.
    
    **Pros**: Maximum visibility, always present
    **Cons**: Takes up screen space
  </Tab>
  
  <Tab title="Footer Only">
    The badge appears only in the footer section of your directory.
    
    **Pros**: Less intrusive, cleaner design
    **Cons**: Only visible when users scroll to bottom
  </Tab>
  
  <Tab title="Disabled">
    No badge is displayed on your directory.
  </Tab>
</Tabs>

## Directory Features

Control which features are available to your visitors to enhance their browsing experience.

### Sorting Tabs

<Check>
Enable **Sorting Tabs** so users can organize listings by:
- **Trending**: Most popular listings based on recent activity
- **Popular**: All-time most viewed or engaged listings
- **New**: Recently added listings
</Check>

This helps visitors discover content in different ways based on their preferences.

### Location Filter

<Card title="Location-Based Search" icon="map-pin">
Activate the **Location Filter** to let users search by location and distance. This is essential for local directories where geographic proximity matters.
</Card>

#### Distance Unit Configuration

Choose the distance unit that suits your audience:

| Unit | Best For | Example Regions |
|------|----------|-----------------|
| **Miles** | US audience | United States, United Kingdom |
| **Kilometers** | International | Europe, Asia, most of the world |

<Tip>
Choose based on your primary audience's location. If you have a global audience, kilometers are the international standard.
</Tip>

### Favorites

<CardGroup cols={2}>
  <Card title="Enable Favorites" icon="heart">
    Allow users to save listings they like for quick access later.
  </Card>
  
  <Card title="Benefits" icon="check-circle">
    - Increases engagement
    - Encourages return visits
    - Helps users organize research
    - Provides data on popular listings
  </Card>
</CardGroup>

### Reviews

<Card title="User Feedback System" icon="star">
Enable **Reviews** so visitors can write and read feedback on listings. This builds trust and provides valuable social proof for listed businesses.
</Card>

**Review features typically include:**
- Star ratings (1-5 stars)
- Written feedback
- Review moderation options
- Response capabilities for listing owners

<Warning>
Remember to moderate reviews to maintain quality and prevent spam or inappropriate content.
</Warning>

## Listings Settings

Control how listings are displayed, submitted, and verified on your directory.

### Listings Per Page

Set the number of listings to display per page:

```plaintext
Recommended values:
- 20-30: Better performance, faster loading
- 40-50: Balanced approach (e.g., 50)
- 60+: More content per page, but slower loading
```

<Info>
Consider your audience's browsing habits and page load performance when choosing this number.
</Info>

### Submission Settings

Control who can interact with your directory and how.

#### Who Can Submit Listings

<Tabs>
  <Tab title="Anyone Without Signup">
    **Open submission** - No registration required
    
    **Pros**: 
    - Lower barrier to entry
    - More submissions
    - Faster growth
    
    **Cons**:
    - Higher spam risk
    - Less control
    - Harder to manage
  </Tab>
  
  <Tab title="Registered Members Only">
    **Gated submission** - Registration required
    
    **Pros**:
    - Better quality control
    - Accountability
    - Build user database
    - Reduce spam
    
    **Cons**:
    - Fewer submissions
    - Extra friction
  </Tab>
</Tabs>

#### Membership Requirements for Viewing

You can require membership to view listings beyond a preview limit:

<Card title="Freemium Model" icon="lock">
This creates a **freemium model** where:
- Visitors can browse a limited number of listings
- Registration unlocks full directory access
- You build an engaged user base
- You can later monetize with premium features
</Card>

### Add New Listing Options

Enhance the listing submission experience with smart features.

#### Website Pre-fill Step

<Card title="Auto-fill Technology" icon="wand-sparkles">

#### AI Content Generation

<Card title="AI-Powered Descriptions" icon="robot">
Enable **AI content generation** to help users create detailed descriptions when AI credits are available.

This is perfect for users who:
- Struggle with writing compelling descriptions

<Info>
AI content generation requires available AI credits. Make sure to monitor your credit usage and refill as needed.

### Links SEO

Configure how search engines treat links from your directory.

<Tabs>
  <Tab title="Do-Follow Links">
    **Pass SEO value** to linked websites
    
    **When to use**:
    - Premium/paid listings
    - Verified businesses
    - Quality directories
    - Incentive for upgrades
    
    **Impact**: Adds value to listings, attracts quality businesses
  </Tab>
  
  <Tab title="No-Follow Links">
    **Don't pass SEO value** to linked websites
    
    **When to use**:
    - Free listings
    - Unverified submissions
    - Spam prevention
    - User-generated content
    
    **Impact**: Reduces spam incentive, protects your site's SEO
  </Tab>
</Tabs>

<Tip>
**Pro strategy**: Use no-follow for free listings and do-follow for premium listings as an upgrade incentive.
</Tip>

### Listing Verification System

Allow users to prove ownership of their listings through one or more verification methods. This builds trust and authenticity.

<Steps>
  <Step title="Choose Verification Methods">
    Select which verification methods to enable on your directory.
  </Step>
  
  <Step title="Configure Visibility Rules">
    Decide whether unverified listings can be visible or not.
  </Step>
  
  <Step title="Communicate Requirements">
    Inform users about verification requirements and benefits.
  </Step>
  
  <Step title="Monitor Verifications">
    Track which listings are verified to maintain directory quality.
  </Step>
</Steps>

#### Verification Methods

<Tabs>
  <Tab title="Email Domain Verification">
    Matches the email domain with the website domain.
    
    **How it works**:
    ```plaintext
    Website: example.com
    Email: owner@example.com
    Result: ✓ Verified (domains match)
    
    Website: example.com
    Email: user@gmail.com
    Result: ✗ Not verified (domains don't match)
    ```
    
    **Best for**: Corporate websites, established businesses
  </Tab>
  
  <Tab title="Business Email Verification">
    Matches the member's email with the business email provided.
    
    **How it works**:
    ```plaintext
    Member Email: owner@example.com
    Business Email: owner@example.com
    Result: ✓ Verified (emails match)
    
    Member Email: user@gmail.com
    Business Email: contact@example.com
    Result: ✗ Not verified (emails don't match)
    ```
    
    **Best for**: Simple verification, quick setup
  </Tab>
  
  <Tab title="Website Code Verification">
    User adds a verification code to their website.
    
    **How it works**:
    1. System generates unique verification code
    2. User adds code to their website HTML
    3. System scans website to confirm code presence
    4. Listing is marked as verified
    
    **Best for**: Maximum security, tech-savvy users
  </Tab>
</Tabs>

#### Multiple Verification Methods

<Card title="Flexible Verification" icon="shield-check">
You can enable **one or more** verification methods to give users options:

- **All three methods**: Maximum flexibility for users
- **Email verifications only**: Simpler, faster process
- **Code verification only**: Highest security standard

Choose based on your directory's needs and your audience's technical capabilities.
</Card>

#### Unverified Listings Visibility

Control what happens to listings that haven't been verified:

<CardGroup cols={2}>
  <Card title="Allow Unverified Listings" icon="eye">
    Unverified listings remain visible with an "Unverified" indicator.
    
    **Pros**: More listings visible, lower barrier
    **Cons**: Less trust, mixed quality perception
  </Card>
  
  <Card title="Restrict to Verified Only" icon="eye-slash">
    Only verified listings are visible to the public.
    
    **Pros**: Higher trust, premium positioning
    **Cons**: Fewer visible listings initially
  </Card>
</CardGroup>

<Warning>
If you restrict visibility to verified listings only, make sure to clearly communicate this requirement during the submission process.
</Warning>

## Configuration Best Practices

<AccordionGroup>
  <Accordion title="Start Simple" icon="seedling">
    Begin with basic settings and add features gradually as your directory grows. Don't overwhelm users with too many options at launch.
  </Accordion>
  
  <Accordion title="Know Your Audience" icon="users">
    Configure settings based on your target users' needs and technical abilities. A tech directory might use code verification, while a local business directory might stick to email verification.
  </Accordion>
  
  <Accordion title="Balance Control and Growth" icon="scale-balanced">
    Requiring registration reduces spam but slows growth. Find the right balance for your directory's stage and goals.
  </Accordion>
  
  <Accordion title="Test Everything" icon="flask">
    After changing settings, test the user experience from a visitor's perspective to ensure everything works smoothly.
  </Accordion>
  
  <Accordion title="Monitor and Adjust" icon="chart-line">
    Track how settings impact user behavior and be willing to adjust based on data and feedback.
  </Accordion>
</AccordionGroup>

## Quick Settings Reference

### Recommended Configurations by Directory Type

#### Local Business Directory
```plaintext
✓ Enable location filter (miles/km based on region)
✓ Enable reviews and favorites
✓ Enable sorting tabs
✓ Email domain or business email verification
✓ Allow unverified listings (with indicator)
✓ 30-50 listings per page
```

#### Professional Services Directory
```plaintext
✓ Enable favorites
✓ Enable sorting tabs
✓ Require registration for submissions
✓ Website code verification (high trust)
✓ Do-follow links for verified/premium
✓ 20-40 listings per page
```

#### SaaS/Tech Directory
```plaintext
✓ Enable all sorting tabs
✓ Enable reviews heavily (social proof important)
✓ Website pre-fill enabled
✓ AI content generation enabled
✓ Website code or email domain verification
✓ 40-60 listings per page
```

#### General Directory
```plaintext
✓ Enable most features (sorting, favorites, reviews)
✓ Open submission (no signup required initially)
✓ Business email verification (simplest)
✓ Show featured in slider
✓ 30-50 listings per page
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Too much spam submissions" icon="shield">
    **Solutions**:
    - Require registration for submissions
    - Enable stricter verification requirements
    - Use manual approval for free listings
  </Accordion>
  
  <Accordion title="Low submission rate" icon="chart-line-down">
    **Solutions**:
    - Remove registration requirement
    - Enable website pre-fill for easier submissions
    - Enable AI content generation
    - Simplify verification process
  </Accordion>
  
  <Accordion title="Users confused about verification" icon="question-circle">
    **Solutions**:
    - Add clear instructions on listing submission page
    - Send follow-up email with verification steps
    - Offer multiple verification methods
    - Provide video tutorial
  </Accordion>
  
  <Accordion title="Page loading slowly" icon="hourglass">
    **Solutions**:
    - Reduce listings per page
    - Disable resource-heavy features
    - Optimize images in slider
    - Consider caching solutions
  </Accordion>
</AccordionGroup>
